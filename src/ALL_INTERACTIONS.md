## 1️⃣ PTO REQUEST WORKFLOW

### **Employee Actions**
- ✅ Click "Request PTO" button → Opens modal
- ✅ Fill form (dates, type, reason)
- ✅ Submit request → Creates pending request
- ✅ **Notification sent to Team Lead & HR Admin** (popup + bell)
- ✅ Cancel pending request
- ✅ View request history

### **Team Lead/HR Admin Actions**
- ✅ **Receive popup notification** when employee requests PTO
- ✅ Bell icon badge shows count
- ✅ See pending approvals section (orange highlight)
- ✅ Click "Approve" → Employee gets notification (popup + bell)
- ✅ Click "Reject" → Enter reason → Employee gets notification (popup + bell)
- ✅ View all requests with filters

### **Notifications Flow**
```
Employee submits → Team Lead/HR Admin notified
Team Lead approves/rejects → Employee notified
```

## 🎯 NOTIFICATION TARGETING

### **By Role**
- `targetRoles: ["team_leader", "hr_admin"]` → Only managers
- `targetRoles: ["employee"]` → Only employees
- No targetRoles → Everyone sees it

### **By Employee ID**
- `targetEmployeeId: 1` → Only employee with ID 1
- `targetEmployeeId: 3` → Only employee with ID 3

### **By Department** (Team Leaders)
- Team Leaders automatically filtered to their department
- Only see notifications from their team
